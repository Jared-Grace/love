Transform examples

Add a two-sided guard to a slot setter

aea prepends arguments_assert_each and repairs imports; one predicate per parameter.

aea js_return_argument_set "js_return_is, js_expression_node_is"
import { property_set } from "./property_set.mjs";
export function js_return_argument_set(r, a) {
  property_set(r, "argument", a);
}
→
import { arguments_assert_each } from "./arguments_assert_each.mjs";
import { js_return_is } from "./js_return_is.mjs";
import { js_expression_node_is } from "./js_expression_node_is.mjs";
import { property_set } from "./property_set.mjs";
export function js_return_argument_set(r, a) {
  arguments_assert_each(arguments, [js_return_is, js_expression_node_is]);
  property_set(r, "argument", a);
}

Flatten a nested call into a named local

atomize lifts each inner call out of its argument slot into a let, so every call reads on its own line. Simplicity is AST depth — one nested expression becomes two shallow statements.

atomize
export function f(a) {
  return g(h(a));
}
→
export function f(a) {
  let v = h(a);
  return g(v);
}

Auto-imports: add missing, remove unused, canonicalize paths

auto composes all three import verbs — list_last (used, unimported) is added, list_first (imported, unused) is removed, list_size (wrong ../../js/ path) is canonicalized to ./. Provably auto-safe: idempotent, total, state-independent.

auto js/mixed_imports.mjs
import { list_first } from "./list_first.mjs";
import { list_size } from "../../js/list_size.mjs";
export function alpha(x) {
  return list_size(x);
}
export function beta(y) {
  return list_last(y);
}
→
import { list_last } from "./list_last.mjs";
import { list_size } from "./list_size.mjs";
export function alpha(x) {
  return list_size(x);
}
export function beta(y) {
  return list_last(y);
}

Repair imports across a whole multi-function file

A migration file can hold many declarations; program-wide repair adds every missing repo import — single-export js_flo would fail here. Locally-declared names (alpha, beta) are never self-imported.

imports js/two_sizes.mjs
export function alpha(x) {
  return list_size(x);
}
export function beta(y) {
  return list_size(y);
}
→
import { list_size } from "./list_size.mjs";
export function alpha(x) {
  return list_size(x);
}
export function beta(y) {
  return list_size(y);
}

Generate a node-type predicate from scratch

ntp creates a new js_<X>_is predicate; 'before' is empty because nothing existed. Runs file-side (sandboxed) — see the app or the temp-file runner.

ntp js_return_is ReturnStatement
(nothing yet — new file)
→
import { js_node_type_is } from "./js_node_type_is.mjs";
export function js_return_is(node) {
  let is = js_node_type_is(node, "ReturnStatement");
  return is;
}

Remove an unused import

prune drops only imports no code references — list_first is imported but never called, so its line goes; list_size stays because alpha uses it. This is the remove-only third of what auto does.

prune
import { list_first } from "./list_first.mjs";
import { list_size } from "./list_size.mjs";
export function alpha(x) {
  return list_size(x);
}
→
import { list_size } from "./list_size.mjs";
export function alpha(x) {
  return list_size(x);
}

A slot setter refuses a statement in an expression slot

The if-test slot holds an expression; a statement in it is refused by the two-sided guard (js_expression_node_is fails on arg 1).

js_statement_if_test_set( parse("if (a) {}"), parseStatement("b();") )
throws — arg 1 is not an expression node
